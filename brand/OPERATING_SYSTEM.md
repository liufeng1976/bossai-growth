# BossAI Operating System (BOS) V1.0

**Status:** Active  
**Version:** 1.0  
**Owner:** BossAI  

This document defines how BossAI operates every day. It is not a vision document. It is an execution system.

---

## 1. Company Goal

North Star Metric is Weekly Completed Work — the number of real business tasks successfully completed by real users each week using BossAI. Examples include generating and publishing a marketing video, finishing a product listing, creating advertising creatives, and managing a customer support workflow. Success is measured by completed work, not feature count.

## 2. Weekly Cycle

Every week follows the same rhythm. Monday is for reviewing last week's metrics, customer feedback, and bugs. Tuesday is for product improvements and UX optimization. Wednesday is for customer interviews and sales demos. Thursday is for QA verification and documentation updates. Friday is for release and the weekly report.

## 3. Decision Rules

Every new feature must answer which customer requested it, which business problem it solves, how success will be measured, and whether it aligns with the Company Constitution. If these questions cannot be answered, the feature stays in Backlog.

## 4. Priority Rules

P0 covers security, data loss, payment, license, and application crash. P1 covers customer blocking issues including installation, API connection, and export. P2 covers high-value customer requests, UX improvements, and workflow optimization. P3 covers nice-to-have features, developer tools, and experimental ideas.

## 5. Team Responsibilities

CEO handles product strategy, business, pricing, and partnerships. Engineering handles architecture, code quality, and testing. Growth handles website, marketing, and brand. Sales handles customer acquisition, demo, and CRM. Customer Success handles onboarding, support, feedback, and case studies.

## 6. Release Policy

Desktop follows stable release only. Cloud follows continuous delivery. Marketplace has an independent release cycle.

## 7. Success Metrics

Weekly metrics include downloads, installations, activated licenses, paying customers, retention, customer satisfaction, completed work, MRR for Cloud, and ARR for Enterprise.

## 8. Roadmap Rules

Version planning must follow Customer Value, then Business Value, then Engineering Effort. Never the opposite.

## 9. Meeting Rules

Every weekly meeting answers only what customers achieved, what blocked customers, and what should be improved next. Not how many commits, how many modules, or how many providers.

## 10. Operating Principle

BossAI exists to help customers complete work. Everything else is secondary.
