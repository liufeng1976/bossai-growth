> 优先级由真实客户需求驱动，非开发者兴趣
> P0 = 阻塞客户使用 / P1 = 重要 / P2 = 锦上添花 / P3 = 未来

## 优先级调整规则

1. 一个客户提出的需求 → 进入 P2 观察
2. 两个客户提出 → 升为 P1 评估
3. 三个及以上客户提出 → 升为 P0/P1 执行
4. 付费客户需求优先级高于试用客户
5. Bug 类（非 Feature）自动进入 P0
6. 每两周根据 weekly-report 重新排序

## 当前 V1.1 开发建议

根据 0 → 10 客户阶段的最常见需求，建议 V1.1 交付顺序：

1. **视频状态轮询**（P0 技术债务）— 当前视频生成后 Director 只显示 taskId
2. **Business Context UI**（P1 最影响销售）— 用户需要可视化配置品牌信息
3. **Workflow Template Selector**（P1 提升演示效果）— 预设工作流更易展示价值
4. **配置持久化统一**（P0 技术债务）— 修复 localStorage 与 IPC 不一致
5. **i18n 即时刷新**（P0 技术债务）— 提升国际用户体验
